微信小程序 学习笔记——初识

小程序开发
  • WXML
  • WXSS
  • 小程序


最近在研究微信小程序开发，从0到1，以官方文件为基础开始学习相关概念。

一、相关概念

<特点>

小程序既不是网页也不是APP，而是融合了二者特点的新应用形态。
其特点有：
1.随时加载、随时使用
2.拥有强大接口和能力
3.易学、易开发

<开发语言>

小程序开发语言有以下几种：

  1. 视图描述语言：
    • WXML
    • WXSS
  2. 逻辑语言：JavaScript（JS)

<开发过程>

  1. 下载、安装开发者工具：mp.weixin.qq.com
  2. 注册小程序账号
  3. 新建小程序项目
  4. 进行开发

二、编写简单小程序

<调整配置文件>

微信小程序中有个文件是 app.json,它功能类似于 IOS/Android App 开发的“清单文件”,记录小程序一些基础配置：

  • 小程序的所有页面路径
  • 小程序全局界面表现
  • Tab标签配置
  • 网络超时
  • 调试模式开启或关闭

<了解小程序获取 用户资料的机制>

在小程序中获取用户资料，需要先在小程序中设置一个按钮，让用户点击；然后建立一个函数，当用户点击该按钮的时候，这个函数将会接收用户资料。这种做法是为了当用户主动要求的时候，小程序再去请求用户资料，避免出现“刚启动小程序就要求授权”的情况

==新建按钮元素==

==从微信获取用户资料==

给<button>组件添加 open-type=getUserInfo 属性，用以声明按钮点击的时间是获取用户资料。
接下来，用JS文件，接收微信传给我们的数据。打开 Hello World.js ，直接删除自动生成的空白函数，只保留 Page() 一个函数。新建一个接收用户资料的函数。

在开发者工具预览区尝试点击按钮授权，点击“同意”后更改 js 文件，第4行改为console.log(res.detail.userInfo)。再次点击授权会看到干净的用户资料。

==将数据处理结果放到界面==

在小程序中，界面层（ WXML ）和逻辑层（ JS ）之间有一种特殊的数据交换方式，名为“数据绑定”。WXML 可以与 JS 中的特殊的变量进行绑定，当 JS 修改变量的时候，变化将会直接传入界面层。在 JS 中，想要将数据放入可在界面层展示的变量，可以利用 Page() 对象本身含有的 setData() 将用户数据写入变量。

在 WXML 中运用“双花括号”的语法，对 JS 中的变量进行绑定。点击按钮，小程序可以获取并展示用户昵称和头像。
